# SSAFY 15기 관통 프로젝트

# 🚀 실시간 금융 댓글 수집 및 AI 증강 서비스 (PJT 05)

본 프로젝트는 Django를 활용하여 특정 금융 자산(주식)의 실시간 커뮤니티 데이터를 수집하고, 이를 Pandas와 LLM(Generative AI)으로 가공하여 사용자에게 제공하는 데이터 파이프라인 웹 서비스입니다.

## 📅 프로젝트 기간
*   **일시**: 2026년 4월 30일
*   **예상 구현 시간**: 필수 기능(6H) + 심화 기능(2H)

## 👥 팀원 및 역할 분담

### **장우창: Data Pipeline & AI Engineer**
*   **Selenium 기반 데이터 수집**: 토스증권 커뮤니티 동적 크롤링 로직 구현 (F103, F104)
*   **데이터 전처리 및 분석**: Pandas를 활용한 IQR 이상치 제거 및 정규표현식 정제 (F105)
*   **LLM 연동**: OpenAI API를 활용한 부적절 댓글 필터링 및 데이터 증강 로직 개발 (F106)
*   **성능 최적화**: 대용량 처리를 위한 **Generator** 기반 데이터 반환 구현

### **황민정: Web Interface & Backend Integration**
*   **웹 서비스 아키텍처**: Django View 로직 및 URL 매핑 구현
*   **실시간 응답 처리**: **StreamingHttpResponse**를 활용한 단계별 처리 현황 실시간 출력 (F107)
*   **사용자 인증 및 입력**: 커스텀 유저 모델 기반 회원가입 및 회사명 입력 검증 (F101, F102)
*   **결과 시각화**: 가공된 원본/정제/증강 데이터 통합 출력 UI 구현 (F107)

## 🛠️ 기술 스택
*   **Framework**: Django 4.2+
*   **Language**: Python 3.9+, HTML5, CSS3
*   **Libraries**: Selenium, Pandas, OpenAI (init_chat_model)
*   **Database**: SQLite3
*   **Tool**: VS Code, Git (Branch Strategy: develop, feature/ 사용)

## ✨ 핵심 기능 (Requirements)

### 1. 사용자 입력 및 검증 (F101)
*   사용자는 웹 화면을 통해 회사명을 입력하며, 미입력 시 데이터 수집 절차가 수행되지 않도록 제한합니다.

### 2. 동적 데이터 수집 및 정제 (F103, F104, F105)
*   **Selenium**을 활용해 토스증권 페이지 접속 및 실시간 댓글 20개를 추출합니다.
*   **Pandas**를 활용해 특수문자 제거 및 IQR(Interquartile Range) 기반의 길이 이상치 데이터 필터링을 수행합니다.

### 3. AI 기반 데이터 증강 및 통합 (F106, F107)
*   **LLM(OpenAI)**을 호출하여 수집된 댓글의 의미를 유지하면서 비슷한 어조의 새로운 댓글을 생성합니다.
*   원본, 전처리, 증강된 데이터를 하나의 결과 집합으로 통합하여 사용자에게 출력합니다.

## 🌿 브랜치 전략
*   **develop**: 메인 개발 브랜치 (최종 병합)
*   **feature/data-pipeline**: Selenium 크롤러 및 AI 전처리 로직 개발 (담당: 장우창)
*   **feature/web-interface**: Django UI 및 스트리밍 응답 구현 (담당: 황민정)

## 📝 느낀 점 및 배운 점

*   **장우창**: 단순히 데이터를 긁어오는 것에 그치지 않고, LLM을 활용해 데이터의 품질을 높이는 과정에서 프롬프트 엔지니어링의 중요성을 배웠습니다. 특히 대량의 데이터를 한 번에 처리할 때 발생하는 부하를 방지하기 위해 **Generator**를 사용하는 법을 익혔습니다.
*   **황민정**: 크롤링과 AI 응답 대기 시간이 긴 작업의 특성상, **StreamingHttpResponse**를 사용하여 사용자에게 실시간 진행 상황을 보여주는 것이 사용자 경험(UX)에 얼마나 큰 차이를 만드는지 체감했습니다. 팀원과 인터페이스(딕셔너리 구조)를 사전에 정의하여 협업 효율을 높일 수 있었습니다.
